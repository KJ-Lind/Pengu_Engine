Frame Loop Data Flow
QTimer::timeout → ViewportWidget::paintGL() → glBindFramebuffer(FBO) → Engine::update() → Renderer::render(scene, cam) → Pipeline::render(scene, cam) → World::getEntitiesWith<T>() → glDrawElements() → EngineUI::render(state, fboTexID)
◆ Qt event loop: app.exec() replaces while(running) — QTimer fires paintGL() at 16ms (~60fps) or 0ms (uncapped)
◆ FBO: glBindFramebuffer(defaultFramebufferObject()) — must restore Qt's internal FBO before UI render, never bind to 0
◆ Resize: resizeGL(w,h) → destroyFBO() + createFBO(w,h) + engine.onResize(w,h) — Qt calls this automatically
◆ UI layer: EngineUI::render(state, fboTexID) — reads/writes EditorState, no Qt headers in Engine
◆ Scene switching: engine.loadScene(newScene) — old scene + all its entities destroyed automatically
◆ Pipeline switching: renderer.setPipeline(make_unique<DeferredPipeline>(rm)) — cleanup() → init() called automatically
◆ Resource ownership: Scene entities hold shared_ptr, ResourceManager holds weak_ptr
